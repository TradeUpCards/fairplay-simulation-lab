Dataset vs. the Labeled Property Graph

What the fairplay-simulation-lab/data fixtures contain today, what the integrity property graph actually needs (our ontology: 7 node types, 3 edge tiers), and the explicit, actionable plan to turn one into the other. Bottom line: the data is rich and correct — but it stores relationships as group objects + scalar signals, while the graph needs individual nodes + pairwise typed edges. The gap is an ETL/explode problem, not a re-collection problem.

The good news. Every node type our ontology needs has a home in the data. The simulator already produces the three demo cases (escalate / monitor-household / benign-overlap), paid-seat-time, pace/volatility, and seeded ground-truth labels. Nothing needs to be re-simulated.
The gap. The graph's core rule — "integrity = convergence of N independent edge-types on one connected component" — needs those signals to be edges. Right now 4 of the 6 signal relationships are aggregate objects (co_seating), per-player scalars (soft_play_delta), or booleans (outsider_pressure_signal). They can't be counted by a WCC convergence gate until they're exploded into (Account)-[:EDGE]->(Account).

04The 3 structural mismatches

Everything above reduces to three recurring shape-mismatches. Fix these patterns and the whole dataset snaps into the graph.

① Groups-as-objects vs. nodes+edges

device_groups · clusters · households

The data models relationships as a container holding player_ids[] (a hyper-edge). The LPG needs a real node (Device / Cluster) plus pairwise / membership edges exploded from that array.

{player_ids:[A,B,C]} → (A)-[:USES_DEVICE]->(D)<-[:USES_DEVICE]-(B)…

② Signals as scalars/booleans vs. typed edges

soft_play_delta · outsider_pressure_signal · co_seating

The discriminating signals live as per-player numbers or per-cluster booleans. The convergence gate can only count them if they're pairwise edges with from/to + strength. A scalar loses the directionality the rule depends on.

③ Evidence & counter-evidence as prose

note · distinguishing_signals

The guardrail requirement — is_counter_evidence, reason_code — has no structured home. "No single signal is proof" is currently an English sentence, not a queryable Evidence node. GraphRAG grounding needs it structured.

(bonus) Thresholds buried in note strings

soft_play ≤ −0.60 · convergence N

Escalation thresholds are written inside note text. Lift them into a small config/thresholds.json so scoring is data-driven and auditable, not parsed from prose.

07ETL pipeline — fixtures → graph

A deterministic loader (Python or TS) that reads the fixtures and emits graph nodes/edges. No new modeling — just explode, derive, and unify.

  1. Load & lock identities
    Read all 7 graph-bound files; freeze player/table IDs; build lookup maps. Fail loudly on any unresolved provisional id.
  2. Emit entity nodes
    players → Account (compute archetype from documented thresholds + risk_score); table_roster → Table; one node per device_group → Device.
  3. Merge Seating
    Join sessions ⨝ seat_events on session_id → one Seating node carrying duration, paid_seat_time, exit_reason + join/leave/hold stamps. Emit PARTICIPATED_IN / AT_TABLE / IN_SESSION.
  4. Explode device groups
    Each device_groups.player_ids[] → N USES_DEVICE edges into the Device node. (Shared device falls out structurally — no signal edge here.)
  5. Derive Tier-2 signal edges
    timing_correlation → TIMING_CORRELATED (rename); co_seating obj → pairwise CO_SEATED_WITH; soft_play / outsider → pairwise edges (needs gaps #1–#2, else derive within-cluster from per-player deltas + flag as inferred).
  6. Unify cases → Cluster nodes
    clusters + households + regular_overlaps → Cluster{status} (escalated / monitor / benign). Explode member_ids → MEMBER_OF. Add CONCERNS_TABLE from members' tables.
  7. Synthesize Evidence
    From pit_boss_evidence_seed + relationship notes + household distinguishing_signals → Evidence{type,score,reason_code,is_counter_evidence} + SUPPORTED_BY. Counter-evidence flag = the household's benign distinguishers.
  8. Run convergence gate
    WCC for components; count distinct Tier-2 edge-types per component vs config/thresholds.json. CASE-C → escalate (device+timing+co-seat+soft-play converge); household → monitor (device only, counter-evidence present); overlap → benign (co-seat only, no device/timing).
  9. Validate against labels
    Assert the graph's status matches seeded_case_labels.expected_* and passes every eval_checks entry. This is your loader's unit test.

08Target schema contract

What the loader must emit — the canonical contract both stacks read from.

// NODES
Account   {account_id, archetype, registered_days_ago, risk_score,
           vpip, pfr, aggression_factor, soft_play_delta, timing_regularity, bot_similarity_score}
Device    {device_id, fingerprint, link_type}
Table     {table_id, stakes, game_type, max_seats, pace_label, volatility_label,
           avg_session_min, paid_seat_time_trend, health_score?, health_band?}
Seating   {seating_id, player_id, table_id, session_id, seated_at, left_at,
           duration_min, paid_seat_time_intervals, exit_reason, status}
Cluster   {case_id, case_type, status:[escalated|monitor|benign], integrity_score}
Evidence  {evidence_id, type, score, reason_code, is_counter_evidence, description}

// EDGES — tier 1 raw
(Account)-[:USES_DEVICE]->(Device)
(Account)-[:PARTICIPATED_IN]->(Seating)-[:AT_TABLE]->(Table)
(Seating)-[:IN_SESSION]->(Session?)

// EDGES — tier 2 signals (window_start/end + strength on each)
(Account)-[:CO_SEATED_WITH {co_seat_count,total_tables,rate}]-(Account)
(Account)-[:TIMING_CORRELATED {correlation,strength}]-(Account)
(Account)-[:SOFT_PLAYS_AGAINST {aggression_delta,hand_count,confidence}]->(Account)   // gap #1
(Account)-[:TARGETS_OUTSIDER {aggression_lift,ev_extracted}]->(Account)              // gap #2

// EDGES — tier 3 case attachment
(Account)-[:MEMBER_OF]->(Cluster)-[:SUPPORTED_BY]->(Evidence)   // gap #3
(Cluster)-[:CONCERNS_TABLE]->(Table)

Note: device_link in the source data is not a graph edge type — it maps to the structural USES_DEVICE path through a shared Device node. Keep the Neo4j UPPER_SNAKE convention for relationship types.

  2. Write the deterministic loader 1–2 days
    The §07 pipeline. Emits nodes + edges from the existing fixtures; flags derived/inferred edges (soft-play/outsider) until gaps #1–#2 land.
  3. Request the 3 high-severity data additions sim team
    Pairwise SOFT_PLAYS_AGAINST (#1), TARGETS_OUTSIDER w/ victim (#2), structured Evidence (#3). These are the only true collection gaps; the rest is transform-on-load.
  4. Add config/thresholds.json + explicit archetype ½ day
    Lift escalation thresholds out of prose; materialize archetype from the documented ID-range / stat thresholds.
  5. Wire the loader's validation to seeded_case_labels ½ day
    Every eval_checks entry becomes a passing assertion — CASE-C escalates, CASE-E monitors, CASE-D stays benign. Green = the graph reproduces ground truth.
  6. Hand the subgraph to GraphRAG after load
    Anchor on Cluster, pull 2-hop subgraph incl. counter-evidence, serialize to triples → grounded pit-boss case summary. Now possible because Evidence is structured.
One-line verdict. The dataset is the right data with the wrong shape for a graph: it stores relationships as group objects and signals as scalars. A single deterministic loader — exploding groups into nodes+edges, deriving pairwise signal edges, and synthesizing Evidence — turns it into a clean LPG. Only three additions need new simulation output (pairwise soft-play, outsider, structured evidence); everything else is transform-on-load.
