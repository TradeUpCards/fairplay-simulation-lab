The P3 Scoring Engine

A deterministic, auditable scoring core. It reads P2's frozen simulation data, computes eight scores with structured reason codes, and freezes the result to JSON the frontend consumes. The LLM is never the detector — structured scoring finds risk; the AI Investigator (P4) only explains it.

✓ Deterministic — same input → byte-identical output 10 archetypes · 4 health terms · 4 integrity bands Python core · stdlib only · → frozen JSON · → TS frontend ✓ All 8 scores shipped & independently audited

05

How the process flows

Per score: read → compute → reason → freeze

  • A build_*.py loads the needed data/*.json fixtures.
  • It runs the pure scoring/*.py functions (which call upstream scores as needed — e.g. health calls integrity for P_clus).
  • Each score emits reason codes {code, detail, signals} — the auditable "why", rendered verbatim by the UI.
  • Output is written to data/derived/, byte-stable across runs.

Validation gate

  • Each validate_*.py asserts the score's pinned acceptance against the seeded answer key.
  • The router validator also asserts the player-facing leakage guardrail (whitelist-subset of safe keys).
  • Every deliverable was independently audited; fixes folded back in.

06

The guardrails the engine enforces

🟢 Player-facing (only this is safe)

  • router_lobby.json → player_lobby — pre-filtered to neutral table facts + a badge.
  • Type LobbyTable is structurally narrowed: no health, rank, risk, or archetype field.
  • Badges: "Recommended for you" · "Good fit" · "Available". Gated tables are absent entirely.

🔴 Operator-facing only

  • Every other derived file + the router operator_view.
  • Numeric scores, archetypes, seating-risk, integrity language — pit-boss console only.
  • seeded_case_labels.json is the answer key — never a player path.
✗ Never states a player cheated ✗ Never recommends an auto-ban / enforcement ⚖ Three separate lenses — health ≠ integrity ≠ promo-abuse ✓ Counter-evidence always surfaced ✓ Always recommends a human action ("elevated for review")
