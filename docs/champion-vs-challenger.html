⚖ How to read this — the two are NOT scored on equal footing

  • Answer key: each player's true archetype is the type P2's generator built it as (documented ID ranges). Both approaches are scored against this same key.
  • Champion (rules): threshold rules reverse-engineered from that same generator, and allowed to read the structural fields (cluster_id, household_id, bot score) that define the rare classes. Scored in-sample — it largely recites the generator rather than learning it.
  • Challenger (ML): one-vs-rest logistic on the 9 behavioral features only (no structural fields), scored out-of-sample by leave-one-out — each player predicted by a model that never saw it. It has to generalize.
  • So "accuracy" here isn't a clean benchmark. The honest read: a model learning from behavior alone lands within ~3 pts of rules that have the answer key baked in. On synthetic rule-generated data this is partly circular; on real data (noisy/missing structural fields, no clean generative rule) the ML's relative value would likely be higher.
Verdict — rules reproduce the generator; the ML generalizes. Overall the rules edge ahead (88.5% vs 85.2%) — but mostly because they get the rare structural classes (new, bot_like, household, cluster) as near-free lookups of the fields that define them, while the ML must infer them from behavior alone and can't learn a 1-example class out-of-sample. On the fuzzy behavioral boundary, where the rule thresholds are arbitrary cuts on a continuum, the ML generalizes better: 81/86 vs 72/86 across recreational/regular/grinder/healthy_anchor. That's the real takeaway — rules for structure, ML for nuance — and why the champion ships the demo while the challenger is promoted only where it measurably helps. The two agree on 79% of players.
